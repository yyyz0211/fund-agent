# 市场情报页 UI 优化规格

## 背景

市场情报页现有 8 个区块平铺展示，信息层次不清，涨跌家数埋在小卡片里缺乏冲击力，板块数据全靠文字颜色区分，缺乏直观性。

## 目标

重新组织页面信息层次，突出核心市场情绪数据，用颜色条形图直观呈现涨跌幅度，提升移动端可读性。

## 信息层次设计

### 第一层 — 市场总览（最醒目）

#### 涨跌情绪横幅
- 大字数字展示上涨/下跌家数（如 `3284 上涨 / 1862 下跌`）
- 配合进度条：蓝色或灰色填充，左侧红色块 + 右侧绿色块，比例直观反映多空力量对比
- 副标签：`偏暖` / `偏弱` / `中性`（根据涨/跌比例判断）
- 涨停/跌停数字作为辅助数据展示在横幅右侧

#### 指数卡片
- 复用现有 `MetricCard`，每个指数一行：指数名 + 点位 + 涨跌幅
- 涨跌幅用背景色区分（绿色 / 红色淡化背景）
- 移动端 2 列，桌面端 4 列

### 第二层 — 板块数据（核心数据）

#### 行业板块
- 涨跌幅 + 资金流向合并为一张表格
- 每行：`板块名 | 涨跌幅(带颜色条) | 净流入(亿)`
- 涨跌幅用内联进度条展示：范围 [-5%, +5%] 映射到固定宽度条形，填充色根据正负使用绿色/红色
- 按涨跌幅绝对值排序，取前 15 条

#### 概念板块
- 同行业板块结构
- 按涨跌幅绝对值排序，取前 15 条

### 第三层 — 背景与延伸

#### 外围市场
- 卡片网格展示：市场名 + 点位 + 涨跌幅
- 港股/美股区域用不同边框色区分（港股蓝色，美股金色）

#### 题材板块
- 保留现有卡片样式，叠加涨跌幅 badge
- 取前 10 条

#### 最新公告
- 保留现有列表样式，增加左侧色条区分（基金公告蓝色系）
- 取前 10 条

### 第四层 — 操作

- 刷新按钮始终固定在页面顶部右侧
- 按钮样式：ghost 风格，不抢占视觉焦点
- 刷新中显示 spinner 状态

## 组件清单

| 组件 | 类型 | 说明 |
|------|------|------|
| `MarketBreadthBanner` | 新增 | 涨跌情绪横幅，含进度条 |
| `SectorTable` | 新增 | 通用板块表格，支持涨跌幅条形 |
| `MarketOverviewCards` | 修改 | 使用 MetricCard 样式，背景色区分涨跌 |
| `OverseasMarkets` | 修改 | 加边框色区分港股/美股 |
| `ThemeBoards` | 修改 | 加涨跌幅 badge |
| `AnnouncementList` | 小改 | 样式微调 |
| `SnapshotRefreshButton` | 修改 | ghost 样式，固定位置 |

## 技术实现

- 使用 Tailwind CSS，不引入新 UI 库
- 涨跌幅条形：固定宽度 `w-24`，根据 pct 映射背景宽度 `w-[${Math.min(Math.abs(pct)/5, 1)*100}%]`
- 移动端响应式：涨跌情绪横幅单列，板块表格横向滚动
